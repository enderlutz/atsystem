# Admin Approval Workflow — Alan & Thomas

## Overview

When the VA processes a new lead, they fill in the fence measurements and the system automatically calculates 3 package prices (Essential / Signature / Legacy). The estimate then lands in the **Admin Approval queue** for you to review, adjust if needed, and send to the client.

---

## Step-by-Step

### 1. Open the Admin Approval Queue

- In the left sidebar, click **Admin Approval** (shield icon — only visible to admins).
- You'll see all pending estimates sorted by urgency:
  - 🔴 **Red** estimates appear first — these flagged items need manual review (unusual pricing, outside service zone, etc.)
  - Then by lead priority: **HOT > HIGH > MEDIUM > LOW**

---

### 2. Review Each Estimate

Each card shows:
- Client name, address, and priority badge
- **Approval badge** (green = auto-approve ready, yellow = add-ons to price separately, red = needs review)
- Three editable price fields: **Essential**, **Signature ★**, **Legacy**

> **Signature** is the recommended middle package — it's pre-highlighted. Clients see the monthly payment for each tier (price ÷ 21 months).

**To preview exactly what the client will see**, click **Preview** — it opens the proposal page in a new tab.

**For full estimate details** (breakdown, inputs, maps link), click **Full Details →**.

---

### 3. Adjust Prices (if needed)

- Edit any of the three price boxes directly.
- The system pre-fills these from the auto-calculated estimate — only change them if you see a reason to.
- Add an optional note in the **Admin notes** field (e.g., "Adjusted for steep terrain").

---

### 4. Send to the Client

Click **Approve & Send** (green button).

This will:
1. Lock in the 3 package prices
2. Generate a unique proposal link
3. Send the client an SMS via GHL with all 3 packages + the proposal link
4. Add a note to their GHL contact with the prices and link

> **Customer hasn't replied yet?** Check **Force send (no reply required)** before approving. Use this only when you're confident in the lead.

---

### 5. Reject (if needed)

Click **Reject**, optionally type a reason, then **Confirm Reject**. The lead is marked rejected and removed from the queue.

---

## What the Client Sees

After you send:
1. Client receives an SMS with all 3 package prices
2. They click the link → proposal page opens
3. They pick their package (Essential / Signature / Legacy)
4. They pick a date from the calendar
5. They confirm their name and address
6. A Google Calendar event is created and you receive a GHL notification

---

## Tips

| Situation | What to do |
|-----------|------------|
| Red estimate | Review the full details, adjust prices manually, then approve |
| Yellow estimate | Send the main fence quote; come back to price add-on services separately |
| Green estimate | Prices are solid — preview if you want, then approve |
| Client hasn't texted back | Use "Force send" — appropriate for warm leads |
| Wrong price calculated | Edit the Signature box (and Essential/Legacy if needed), then approve |
| Bad lead / not serviceable | Reject with a brief note |

---

## Quick Reference

- **Admin Approval** tab → pending queue
- **Preview** button → see the customer proposal before sending
- **Full Details** → full estimate breakdown + inputs + maps
- **Force send** checkbox → bypass "must reply first" guardrail
- SMS + proposal link sent automatically on approve
